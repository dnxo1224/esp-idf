# ESP32 CSI 데이터 수집 프로젝트 — 방향성 정리

## 1. 프로젝트 목적

ESP32를 이용해 **Wi-Fi CSI(Channel State Information)** 데이터를 수집하고,
이를 기반으로 **실내 공간에서의 사람 행동 인식 / 위치 추정**을 수행하는 것이 최종 목표다.

CSI는 송신기(Tx)에서 수신기(Rx)로 가는 무선 채널의 서브캐리어별 진폭·위상 정보를 담고 있어서,
사람이 그 사이를 지나가거나 움직이면 채널이 변하고, 그 변화 패턴으로 행동/위치를 분류할 수 있다.

## 2. 측정 환경 (공간 구성)

- **정사각형 형태의 측정 공간**을 사용한다.
- 공간 한가운데에 **Tx(송신기) 1대**가 위치한다.
- **Rx(수신기) 8대**를 다음과 같이 배치한다:
  - 정사각형의 **네 꼭짓점에 4대**
  - 정사각형의 **네 변의 중심점에 4대**
  - → 총 **8개 Rx** (Rx1 ~ Rx8)
- 측정 공간(정사각형)을 다시 **4개의 정사각형 구역(Zone 1~4)** 으로 나누어,
  사람이 각 구역에 위치한 상태로 데이터를 수집한다.

```
          변중심          변중심
    꼭짓점 ───── ●(Rx) ───── 꼭짓점
      │      Zone?  │  Zone?      │
 변중심●        [ Tx ]         ●변중심
      │      Zone?  │  Zone?      │
    꼭짓점 ───── ●(Rx) ───── 꼭짓점
          변중심          변중심
```

## 3. 수집 시나리오 (라벨 구조)

수집되는 데이터는 다음 차원의 조합으로 라벨링된다.

| 차원 | 값 | 설명 |
|------|-----|------|
| 대상자(PERSON) | 이니셜 (예: `swt`, `kjh`, `kjh_swt`) | 1명 또는 2명(이니셜 결합) |
| 행동(ACTION) | 3가지 (예: `walk`, `sit`, `stand`/`handsup` 등) | 구역마다 동일 행동 집합 |
| 구역(ZONE) | `1` ~ `4` | 정사각형을 4분할한 구역 |
| 수신기(RX) | `rx1` ~ `rx8` | 8개 Rx 동시 수집 |

- **1인 시나리오**와 **2인 시나리오**를 모두 수집한다 (2인은 이니셜을 결합해 표기, 예: `kjh_swt`).
- 즉 한 번의 측정 세트 = (대상자 × 행동 × 구역)에 대해 **8개 Rx CSV**가 동시에 생성된다.

## 4. 송수신 방식

- Tx는 **임의의 MAC 주소로 broadcast ping**을 쏜다.
- 각 Rx는 이 패킷을 수신하면서 CSI를 추출해 시리얼로 내보내고, PC가 CSV로 저장한다.
- 목표 수신율은 **약 30Hz** (초당 30프레임).

## 5. 핵심 문제점 — Rx가 Tx를 가릴 때 수신율 급락

- Tx 신호의 **직진성(LOS 의존성)이 매우 강하다.**
- 사람이나 물체가 **Tx–Rx 사이의 직선 경로(LOS)를 막아 NLOS 환경으로 바꾸면**
  해당 Rx의 수신율이 급격히 떨어진다.
- 예시: 사람이 **Zone 1 정중앙**에 서면, 30Hz 기준 **Rx1 수신율이 16Hz까지** 하락.

이 문제 때문에 일부 데이터 세트는 특정 Rx의 수신율이 낮은 상태로 수집된다.

## 6. 지금 풀어야 할 질문

> **수신율이 낮게 수집된 데이터를, 학습/분석에 그대로 써도 되는가?**

수신율(Hz)이 낮다는 사실만으로는 폐기 여부를 단정할 수 없다. 판단의 핵심은:

- **유실이 "균일하게 다운샘플링"된 것인가** → 프레임 간격이 고르게 벌어진 형태라면,
  단순히 낮은 샘플레이트일 뿐 CSI 신호 구조(행동 패턴)는 보존될 수 있어 **사용 가능**할 수 있다.
- **유실이 "구간 단위 블랙아웃(burst)"인가** → 특정 시간 구간이 통째로 비면
  그 구간의 행동 정보가 사라져 **사용 곤란**하다.

따라서 단순 Hz 수치가 아니라,
**프레임 간격 분포 / 시간축 수신율 / CSI 진폭 패턴의 보존 여부**를 함께 봐야 한다.

## 8. 데이터 포맷 메모 (CSV)

- 헤더 없음, 콤마 구분.
- `col[0] = "CSI_DATA"`
- `col[2] = seq` (송신 시퀀스 번호, 정상이면 1씩 증가 → 유실 추적의 핵심)
- `col[4] = rssi`
- `col[25 : -1] = CSI raw I/Q 배열` (`Q,I,Q,I...` 정수)
- `col[-1] = 수집기가 붙인 절대 수신시각(epoch, float)`
- 깨진 행: 한 줄에 두 프레임이 붙거나(`col[8]`에 또 `CSI_DATA`), 따옴표/대괄호가 섞임 → 파싱 시 필터링.
